## 🛠️ Example Profile Configuration

Create a file named `winaso_profile.json` in the installation directory to pre-set your preferences:

```json
{
  "scanDepth": "deep", 
  "heuristicsLevel": "aggressive",
  "backupStrategy": "differential",
  "languages": ["en", "de", "ja"],
  "exclusions": [
    "HKEY_LOCAL_MACHINE\\SOFTWARE\\Adobe",
    "HKEY_CURRENT_USER\\Software\\Microsoft\\OneDrive"
  ],
  "autoRepair": false,
  "notificationChannel": "email",
  "useCloudBackup": false
}
```

This configuration instructs the tool to perform a deep scan with aggressive heuristics (ideal for heavily cluttered registries), exclude Adobe and OneDrive branches to avoid breaking licensed software, and store backups locally only.

## 💻 Example Console Invocation

For headless or remote deployments, use the command-line interface:

```
winaso-cli.exe --scan --profile "C:\Configs\winaso_profile.json" --output "C:\Reports\registry_audit_report_{timestamp}.json" --quiet
```

This command triggers a scan with the custom profile, saves a timestamped JSON report, and suppresses GUI pop-ups. On completion, the tool fires a system event (Event ID 1102) that your monitoring service can hook into.
